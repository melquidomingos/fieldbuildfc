1 Pré-requisitos

Criar conta no GitHub

O GitHub é onde o arquivo ficará armazenado e publicado. Se você já tem conta, pule para a seção 2.

1
Acesse github.com
Abra o navegador e acesse https://github.com
2
Clique em "Sign up"
No canto superior direito da página, clique no botão Sign up (ou "Criar conta" se estiver em português).
3
Preencha o cadastro
Informe seu e-mail, crie uma senha e escolha um nome de usuário. Anote o nome de usuário escolhido — ele fará parte do endereço do seu sistema.
ℹ️
Exemplo
Se você escolher o nome de usuário joaosilva, o link do sistema será:
https://joaosilva.github.io/fieldbuildfc
4
Confirme o e-mail
O GitHub enviará um código de verificação. Abra seu e-mail, copie o código e cole na tela do GitHub para ativar a conta.
✅
Conta criada com sucesso!
Você verá a tela inicial do GitHub com seu perfil. Prossiga para o próximo passo.
2 Repositório

Criar o repositório do projeto

Um repositório é a "pasta" do GitHub onde os arquivos do projeto ficam armazenados.

1
Clique no botão "New" ou "+"
Na tela inicial do GitHub, clique no botão verde New (canto superior esquerdo) ou no ícone + no canto superior direito e selecione "New repository".
2
Configure o repositório
Preencha os campos conforme abaixo:
CampoValor
Repository namefieldbuildfc (tudo em minúsculas, sem espaços)
DescriptionSistema de Acompanhamento de Execução de Obras
Visibility✅ Marque Public — obrigatório para GitHub Pages gratuito
Add a README file✅ Marque esta opção
Add .gitignore / LicenseDeixe como "None" (pode ignorar)
3
Clique em "Create repository"
Clique no botão verde Create repository no final da página. Você será redirecionado para a página do repositório criado.
✅
Você verá a pasta do projeto com um arquivo README.md já dentro.
3 Upload do arquivo

Fazer upload do arquivo

Agora você vai enviar o arquivo fieldbuildfc.html para o repositório. Para o GitHub Pages funcionar, o arquivo principal precisa se chamar index.html.

⚠️
Renomeie o arquivo antes de fazer o upload
Antes de enviar ao GitHub, renomeie o arquivo de fieldbuildfc.html para index.html. Isso faz com que o GitHub Pages o sirva diretamente pelo link raiz do projeto, sem precisar digitar o nome do arquivo na URL.
1
Baixe o arquivo fieldbuildfc.html
Baixe o arquivo pela conversa do Claude (botão de download no card do arquivo) e salve no seu computador. Em seguida, renomeie-o para index.html.
2
Na página do repositório, clique em "Add file"
Você verá um botão Add file logo acima da lista de arquivos. Clique nele e selecione "Upload files".
3
Arraste ou selecione o arquivo
Arraste o arquivo index.html para a área pontilhada, ou clique em "choose your files" para abrir o seletor de arquivo.
4
Confirme o commit (envio)
Após o arquivo aparecer na lista, desça a página até a seção "Commit changes". Deixe a mensagem padrão ou escreva algo como Adiciona sistema FieldBuildFC. Certifique-se de que "Commit directly to the main branch" está selecionado.
mensagem de commit sugerida
Adiciona sistema FieldBuildFC v1.0
Clique em Commit changes.
✅
O arquivo index.html aparecerá na lista de arquivos do repositório.

Ao final desta etapa, a estrutura do repositório será:

📁 fieldbuildfc/
├── index.html   ← o sistema completo
└── README.md    ← criado automaticamente pelo GitHub
4 GitHub Pages

Ativar o GitHub Pages

Esta etapa transforma o repositório em um site público acessível por qualquer navegador.

1
Abra as configurações do repositório
Na página do repositório, clique na aba Settings (ícone de engrenagem, última aba do menu horizontal).
2
Acesse a seção "Pages"
No menu lateral esquerdo de Settings, desça até a seção "Code and automation" e clique em "Pages".
3
Configure a fonte de publicação
Em "Build and deployment", configure assim:
CampoValor a selecionar
SourceDeploy from a branch
Branchmain
Folder/ (root)
4
Clique em "Save"
Clique no botão Save ao lado da configuração de branch. O GitHub iniciará automaticamente o processo de publicação (leva de 1 a 3 minutos).
⏳
Aguarde o deploy
Você pode acompanhar o progresso acessando a aba Actions no menu do repositório. Uma bolinha laranja indica que está processando; verde significa que está no ar.
✓
Confirme o link gerado
Após o deploy concluir, volte para Settings → Pages. Você verá uma caixa verde com o texto "Your site is live at" e o link do sistema.
5 Acesso público

Acessar o sistema pelo link

Com o deploy concluído, o sistema estará disponível publicamente na internet.

🎉 Seu link será no formato:

Substitua seu-usuario pelo seu nome de usuário do GitHub:

https://seu-usuario.github.io/fieldbuildfc

Exemplo real: https://joaosilva.github.io/fieldbuildfc

📱
Acesse de qualquer dispositivo
Este link funciona em celular, tablet e desktop — em qualquer navegador, sem instalação. Envie o link por WhatsApp para a equipe de campo.
💾
Dados salvos localmente em cada dispositivo
Os dados (obras, atividades, fotos) ficam salvos no localStorage do navegador de cada dispositivo. Cada usuário tem seus dados próprios no aparelho dele. Para centralização real dos dados em nuvem seria necessário um backend — mas para testes e validação, o sistema funciona completamente.
6 Atualizações

Como atualizar o sistema no futuro

Quando receber uma versão nova do sistema (por exemplo, novas funcionalidades ou correções), siga este processo:

1
Baixe o novo arquivo do Claude
Salve o arquivo no computador e renomeie para index.html.
2
Acesse o repositório no GitHub
Entre em github.com/seu-usuario/fieldbuildfc e clique sobre o arquivo index.html na lista.
3
Clique no ícone de edição (lápis)
No canto superior direito da visualização do arquivo, clique no ícone de lápis ✏️ ou no botão com três pontinhos ··· e selecione "Upload file". Isso substitui o arquivo antigo pelo novo.
⚠️
Alternativa mais simples — substituir via upload
Na página do repositório, clique em Add file → Upload files. Envie o novo index.html. O GitHub substituirá automaticamente o arquivo de mesmo nome.
4
Confirme o commit
Escreva uma mensagem descritiva (ex: Atualiza para versão 1.1 — corrige exportação) e clique em Commit changes. O site será atualizado automaticamente em 1–2 minutos.
🕐
Cache do navegador
Se após a atualização o sistema parecer igual ao anterior, pressione Ctrl + Shift + R (Windows/Linux) ou Cmd + Shift + R (Mac) para forçar o recarregamento sem cache.
7 PWA — Celular

Instalar como app no celular

O FieldBuildFC pode ser instalado como um aplicativo na tela inicial do celular, funcionando como um app nativo — inclusive offline.

📱
No iPhone (Safari)
1. Abra o link do sistema no Safari
2. Toque no ícone de compartilhar (quadrado com seta para cima, na barra inferior)
3. Role a lista e toque em "Adicionar à Tela de Início"
4. Confirme tocando em "Adicionar"
🤖
No Android (Chrome)
1. Abra o link no Chrome
2. Toque nos três pontos (⋮) no canto superior direito
3. Toque em "Adicionar à tela inicial" ou "Instalar aplicativo"
4. Confirme tocando em "Adicionar"
📶
Uso offline no canteiro de obra
Após instalar e abrir o app ao menos uma vez com internet, os dados do localStorage permanecem disponíveis mesmo sem conexão. Atividades cadastradas offline ficam salvas no dispositivo e podem ser sincronizadas quando houver sinal.

❓ Dúvidas Frequentes

FAQ

O GitHub Pages é gratuito para sempre?
Sim. Para repositórios públicos, o GitHub Pages é gratuito sem limite de tempo. O único requisito é que o repositório seja público.
Os dados ficam salvos se eu fechar o navegador?
Sim. Os dados ficam salvos no localStorage do navegador do dispositivo. Eles persistem enquanto o usuário não limpar os dados do navegador. Cada dispositivo tem seu próprio banco local.
Posso usar um domínio próprio (ex: sistema.minhaempresa.com.br)?
Sim. Em Settings → Pages, há a opção "Custom domain" onde você pode apontar um domínio próprio. É necessário configurar o DNS do domínio para apontar para os servidores do GitHub (as instruções aparecem na própria tela do GitHub).
O sistema funciona sem internet no canteiro?
Sim, após o primeiro acesso com internet. O sistema salva tudo localmente. Atividades cadastradas offline ficam armazenadas no dispositivo e o badge de "Sincronizar" aparece quando a conexão voltar.
O código-fonte do sistema fica público no GitHub?
Sim, em repositórios públicos o código-fonte é visível. Se preferir manter o código privado, você precisa de uma conta GitHub Pro (paga) para usar GitHub Pages com repositórios privados — ou hospedar em outra plataforma como Netlify ou Vercel, que oferecem deploy gratuito de repositórios privados.
